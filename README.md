# PX4 ULog Analyzer 🛸 飛行日誌視覺化分析儀表板

> **極致流暢、無須安裝、資料不外傳的專業無人機飛行日誌分析工具。**

本專案是一個 100% 純前端（Client-side）的無人機 ULog 日誌分析儀表板。所有日誌解析與渲染均在本地瀏覽器（Web Worker & WASM）中完成，保證飛行數據的絕對隱私與安全。

### 🚀 [點此開啟線上分析儀表板 (Live App Demo)](https://labujuice.github.io/HuLogAnalyzer/dist/)

---

## 🌟 核心功能特色

### 📊 1. 高效能時序數據圖表 (uPlot)
* **大數據降採樣對齊 (LTTB)**：在 ULog 數據量極大時，採用 LTTB 演算法對時間軸及各欄位同步降採樣，徹底解決數據點數不一導致的縮線 Bug。
* **高流暢互動**：支援滑鼠滾輪縮放、中鍵拖曳平移、方向性左鍵拖曳（左拉放大、右拉縮小）、快速鍵一鍵重置視角（`Esc` / `R`）。
* **實時數值懸浮框**：完全繞過 React 渲染流程，直接以 DOM 驅動的數據 Tooltip，保證 60 FPS 級別的懸浮對齊流暢度。
* **多維度欄位拖曳**：支援 `Ctrl+Click` 與 `Shift+Click` 複選欄位，可成批拖曳進圖表中自動對齊並插值。

### 🛸 2. 3D 實時姿態與軌跡觀測器 (Three.js)
* **三維模型動態回放**：以 Three.js 動態建立高質感四軸飛行器模型，播放時實時解算四元數姿態進行三維旋轉。
* **起飛點為原點**：讀取日誌本地位置第一點作為相對原點 `(0, 0, 0)`，自動將 NED (北-東-地) 座標平移映射至 WebGL 空間。
* **歷史/未來雙線軌跡**：
  * **🔴 紅色實線**：代表已經飛過的歷史航線。
  * **🟡 黃色實線**：代表即將飛行的未來預計航線。
* **跟隨相機**：鏡頭焦點鎖定無人機平滑平移，並支援按住左鍵任意旋轉視角與滾輪縮放。

### ✈️ 3. AHRS 航空水平儀 HUD (Canvas 2D)
* **主飛行指示器**：在圓形遮罩下以 HTML5 Canvas 實時渲染航空級水平儀儀表。
* **刻度精準顯示**：包含動態傾斜指針、跟隨 Pitch 俯仰平移的俯仰天梯（刻度標記與朝向角），以及實時 Roll & Pitch 數位面板。

### 🗺️ 4. 2D GIS 地圖軌跡追蹤 (Leaflet)
* **無須 API Key 的底圖**：整合 Google 的衛星混合圖 (Satellite Hybrid) 與向量道路圖 (Roadmap)，隨時一鍵切換。
* **雙色路徑劃分**：
  * **🔴 紅色實線**：代表已飛過的歷史軌跡。
  * **🟡 黃色虛線**：代表尚未飛過的未來預計軌跡。
* **自動追蹤與鎖定**：具備「鎖定跟隨」與「完整視野縮放」切換，指針圖標根據偏航角（Yaw）實時旋轉。

### ⚡ 5. 極致流暢時脈與零 React 負載
* **時間發佈器機制 (`timePublisher`)**：建立純 JS 的高頻時間事件訂閱機制。播放時直接驅動 DOM 元素、uPlot、Three.js 與 Leaflet，**完全跳過 React 的 Virtual DOM 比對**，大幅降低 CPU 負載。
* **高精準時脈**：修正 React 渲染延遲對播放 RAF 迴圈的拖曳現象，播放速度精準契合物理時間並支援 0.25x ~ 10x 倍速。

## 🛠️ 本地開發與編譯

### 1. 安裝環境
本專案基於 **Node.js**、**Vite** 與 **TypeScript** 建置。請先複製專案並安裝依賴：
```bash
# 安裝相依套件 (包括 Three.js、uPlot、Leaflet 等)
npm install
```

### 2. 啟動本機開發與測試伺服器
* **僅啟動開發伺服器 (即時熱重載 HMR)**：
  ```bash
  npm run dev
  ```
  啟動後可在瀏覽器開啟 `http://localhost:5173/` 進行即時開發偵錯。

* **僅啟動編譯產物預覽伺服器**：
  ```bash
  npm run preview
  ```
  啟動後可在瀏覽器開啟 `http://localhost:4173/` 測試已打包的 `dist` 靜態部署檔。

* **一鍵同時啟動雙伺服器 (開發 + 靜態預覽)**：
  ```bash
  npm run serve:all
  ```
  該指令會同時在背景執行 Dev 伺服器 (`:5173`) 與 Preview 伺服器 (`:4173`)，方便您一邊修改原始碼，一邊測試與比對打包後的靜態網頁效能。

* **關閉所有測試伺服器**：
  ```bash
  npm stop
  ```
  當您使用背景執行或 `serve:all` 啟動伺服器後，可以執行此指令一鍵釋放並關閉 Port `5173` 與 Port `4173` 的所有背景程序。

### 3. 編譯為生產環境靜態網頁
```bash
npm run build
```
編譯完成後，所有的靜態網頁產物均會輸出至 **`dist/`** 資料夾。
* 由於配置了相對路徑 `base: './'`，您可以直接雙擊 `dist/index.html` 進行本機離線開啟，或者直接將 `dist/` 上傳至 GitHub Pages 或任何靜態主機。

## 📝 更新日誌與合併管理規範 (Update Log Rules)

為了維持專案更新歷程的清晰性，本專案採用 `UPDATE_LOG.md` 管理每次代碼合併（Merge Request）前的異動項目。不論是**人類開發人員**或 **AI Coding Agent** 均須嚴格遵守以下操作規範：

### 1. 更新日誌路徑
所有異動記錄必須寫入專案根目錄下的 `UPDATE_LOG.md` 檔案中。

### 2. 分支開發規範 (Branch Development)
* 當從 `main` 分支切出新分支進行開發時（例如當前分支 `0704_optimize` 係由 `main` 分支 `2022367` 節點切出）：
  * 必須在 `UPDATE_LOG.md` 檔案的最頂端建立一個全新的最新區塊。
  * 區塊標題格式必須為：`## [Branch: 分支名稱] (基於 main 分支 CommitID 節點切出)`。
  * 於該分支進行的任何調整與功能新增，皆須同步在該區塊的 `更新項目明細` 列表中進行更新。

### 3. 主分支直接提交規範 (Direct Commits on `main`)
* 如果是直接在 `main` 分支上進行調整或修復：
  * **每一次的 Commit** 都必須同時修改並更新 `UPDATE_LOG.md`，在最頂端的主分支紀錄區塊（或新增對應日期的區塊）中加上本次 Commit 調整或新增項目的詳細說明。

### 4. 給 AI Agent 的特別指令 (Instructions for LLM / AI Agents)
* **自動讀取**：AI Agent 在進行任何代碼修改前，應優先讀取 `UPDATE_LOG.md` 以了解目前的更新進度與架構變動。
* **自動更新**：在確認或交付工作前，AI Agent 應自動執行 `UPDATE_LOG.md` 的寫入，並根據當前運作的分支（可透過 `git status` 或 `git branch` 取得）寫入正確的標記格式，最後連同代碼一起提交 Commit。
