# 專案需求與設計規格書：進階日誌分析與模組化客製運算功能 (2026-07-05 修正版)

## 1. 專案背景與擴充目標 (Project Overview)

本需求文件為純前端 PX4 ULog 分析儀表板之進階後處理與分析功能的規格修正版。基於先前討論與使用者反饋，本次功能擴充包含五大分析模組、自訂時序對齊、側邊欄快捷繪圖區（按需計算）以及自定義運算導入（模組化配置），藉以支援深度飛行診斷。

---

## 2. 核心分析功能設計 (Core Analysis Features)

### 2.1 振動量頻譜分析 (Vibration FFT Analysis - Freq vs. Amplitude)
* **呈現圖表：** 頻率 (Frequency, Hz) vs. 幅值 (Amplitude)。
* **多感測器獨立顯示：** 
  * 加速度計 (Accelerometer)：`sensor_accel:0`（三軸 X, Y, Z 各自呈線）。
  * 陀螺儀 (Gyroscope)：`sensor_gyro:0`（三軸 X, Y, Z 各自呈線）。
* **計算公式：**
  * 去直流分量 (De-trending)：$x_i' = x_i - \bar{x}$。
  * 加漢寧窗 (Hanning Window) 降低頻譜洩漏：$x_i'' = x_i' \cdot 0.5 \left[1 - \cos\left(\frac{2\pi i}{N-1}\right)\right]$。
  * 離散傅立葉轉換 (FFT) 後，振幅計算公式為：
    $$\text{Amplitude}(f) = \frac{2 \cdot |X(f)|}{N} \quad (f > 0)$$
    $$\text{Amplitude}(0) = \frac{|X(0)|}{N}$$
* **性能優化：** 
  * 採**按需計算 (Compute-on-demand)** 策略。只有在建立或選取該分析面板，且按下計算時，Worker 才執行 FFT 運算。
  * 支援時間框選 (Time selection) 解析，自動載入框選範圍內數據進行 FFT（點數補零或截斷至 $2^n$ 點，以利 Radix-2 演算法）。

### 2.2 多層級控制器響應與同軸對比分析 (Multi-level PID Controller Response & Setpoint/Feedback Tracking)
比對「期望指令 (Setpoint)」與「實測回饋 (Actual/Estimated)」，涵蓋以下四個主要控制環路：

| 控制層級 | 期望值資料源 (Setpoint Topic & Fields) | 實測值資料源 (Actual Topic & Fields) |
| :--- | :--- | :--- |
| **角速度 (Rate)** | `vehicle_rates_setpoint`<br>• `roll`, `pitch`, `yaw` | `vehicle_angular_velocity`<br>• `xyz[0]`, `xyz[1]`, `xyz[2]` |
| **姿態角 (Attitude)** | `vehicle_attitude_setpoint`<br>• 尤拉角轉換或 `q_d[4]` | `vehicle_attitude`<br>• 四元數 `q[4]` 轉 Roll, Pitch, Yaw |
| **速度 (Velocity)** | `vehicle_local_position_setpoint`<br>• `vx`, `vy`, `vz` | `vehicle_local_position`<br>• `vx`, `vy`, `vz` |
| **位置 (Position)** | `vehicle_local_position_setpoint`<br>• `x`, `y`, `z` | `vehicle_local_position`<br>• `x`, `y`, `z` |

本模組拆分為兩大呈現方式，皆支援對齊與時域修正：

1. **同軸時序追隨對比圖 (Setpoint & Feedback Synced Plotting)**：
   * **分軸處理 (Axis Splitting)**：為了避免將多軸數據混在同一圖表導致混亂，圖表必須按物理軸向（Roll、Pitch、Yaw，或 X、Y、Z 軸）進行**分軸獨立繪製**。
   * **呈現方式**：在同一個畫框內，分軸顯示期望值（Setpoint，如綠色虛線）與實際值（Feedback，如紅色實線）的對比。使用者可以透過分頁或子圖表，分開觀察 Roll、Pitch、Yaw 的跟隨情況。
2. **控制器響應評估 (Controller Response Analysis & Metrics)**：
   * **分軸評估 (Axis-specific Evaluation)**：專注於量化追隨性能，並依物理軸向（Roll、Pitch、Yaw，或 X、Y、Z 軸）分別計算與繪製各自的「偏差誤差 (Error Offset)」時序曲線。
   * **分軸輸出指標**：對各軸獨立計算並輸出以下指標：
     * **分軸均方根誤差 (RMSE)**：評估該軸向的絕對追隨偏差。
     * **分軸皮爾森相關係數 (Pearson R)**：評估該軸向動態響應的相位與趨勢一致性。
     * **分軸相位延遲 (Phase Lag)**：估算該軸向毫秒級的控制反應時間差。

* **數據對齊與時鐘抖動校正：**
  1. **線性插值對齊 (Time Interpolation)**：以實測值時間戳 $t_{\text{ref}}$ 為基準，將期望值插值至該時間點。
  2. **延遲校正與時間對齊 (Jitter & Skew Correction)**：
     * 對於具有時間偏置 (Time offset) 的主題，提供一個簡單的互相關 (Cross-correlation) 延遲偵測，求出使兩序列相似度最高的 $\Delta t$，在插值前先對 Setpoint 時間戳進行平移補償：
       $$t_{\text{setpoint}}' = t_{\text{setpoint}} - \Delta t$$

### 2.3 馬達輸出平衡與電調分析 (Motor & ESC Analysis)
* **馬達軸數自適應**：動態檢測 ULog 內的 `actuator_outputs` 與 `esc_status` Channel 數量（自動適配 4, 6, 8 等多旋翼佈局）。
* **分析輸出**：
  1. **ESC Telemetry 轉速回饋**：若存在 `esc_status`，繪製各馬達 RPM 時序圖。
  2. **Actuator Outputs 命令輸出**：繪製 `actuator_outputs.output[i]` 的時序命令圖（PWM 脈寬或 DShot 數值）。
  3. **Hover 平衡偏離率柱狀圖**：在懸停狀態下，計算各馬達相對於總均值的離差 $\text{Deviation}_j$，用以分析硬體結構是否歪斜或槳葉受損。

### 2.4 磁場強度與航向角分析 (Magnetic Field & Heading Analysis)
1. **磁場模長分析 (Magnetic Vector Norm)**：
   * 讀取磁力計數據（如 `vehicle_magnetometer` 欄位 `magnetometer_ga[3]` 或是 `sensor_mag`）。
   * 計算三軸合成磁場強度模長（單位：Gauss 或 Tesla）：
     $$\text{Mag Norm} = \sqrt{B_x^2 + B_y^2 + B_z^2}$$
   * 用於檢測大電流產生的電磁干擾 (EMI)。
2. **航向對比分析 (Heading Comparison & Alignment)**：
   * **EKF 融合航向 (EKF Estimated Yaw)**：從 `vehicle_attitude` 的四元數 $q$ 計算主要 Yaw 角：
     $$\text{Yaw}_{\text{est}} = \operatorname{atan2}\left(2(q_0 q_3 + q_1 q_2), 1 - 2(q_2^2 + q_3^2)\right)$$
   * **EKF GSF 航向 (EKF GSF Yaw)**：讀取 EKF GSF (Gaussian Sum Filter) 航向估測器 Topic `yaw_estimator_status`（例如 `yaw` 欄位）。當磁力計受干擾或失效時，GSF 利用 GPS 與 IMU 獨立估算航向，將此與 EKF 融合航向重疊對比，能有效診斷指南針異常與初始化偏置。
   * **測量與外部航向 (Measured Yaw & GPS COG)**：
     * 比對 GPS 航向角 `sensor_gps.cog_rad` (Course Over Ground)。
     * 或利用傾角補償後的磁力計讀值進行磁航向估算。
   * **多源航向對比圖**：在一張圖表中同時繪製 EKF Yaw、EKF GSF Yaw、Magnetometer Yaw、GPS COG，進行多源對比。

### 2.5 飛行狀態與模式分析 (Flight Status & Modes Analysis)
為幫助使用者快速掌握飛行狀態，降低手動過濾 Topic 的時間，本模組彙整並解碼以下關鍵指標：

1. **目前飛行模式與解鎖狀態 (Flight Mode & Arming State)**：
   * **Topic 與欄位**：`vehicle_status` (欄位 `nav_state` 與 `arming_state`)。
   * **功能**：讀取數值並將其映射為人類可讀字串（如 `POSCTL`, `AUTO_MISSION`, `ARMED`），並在圖表中以色帶或標籤標記模式變更的時間點。
2. **遙控器搖桿輸入與介入 (Manual Stick Input & Pilot Intervention)**：
   * **Topic 與欄位**：`manual_control_setpoint` (欄位 `x` (Pitch), `y` (Roll), `z` (Throttle), `r` (Yaw))。
   * **功能**：觀察操縱桿 $[-1, 1]$ 區間變化。當搖桿有明顯動作時，表示「使用者介入操作」，可用於輔助判斷姿態突變是來自人為控制還是外界干擾。
3. **系統故障與估算器錯誤 (System Faults & Estimator Errors)**：
   * **Topic 與欄位**：`estimator_status` (如 `control_mode_flags`, `gps_check_fail_flags`, `filter_fault_flags` 等位元遮罩) 以及系統 `L` 文字日誌。
   * **功能**：解碼估算器錯誤旗標，在時序圖上標記 EKF 故障（如 magnetometer check fail、velocity check fail 等）或感測器失效時間點。
4. **安全防護觸發狀態 (Failsafe Triggers)**：
   * **Topic 與欄位**：`vehicle_status` (欄位 `failsafe`, `rc_signal_lost`) 與 `battery_status` (欄位 `warning`)。
   * **功能**：直觀顯示是否觸發 Failsafe（如低電量返航、遙控訊號中斷失控保護）。

## 3. 側邊欄版面與交互設計 (UI/UX layout)

側邊欄 (Sidebar) 改為上下兩區塊設計，強化快速分析的便利性：

```
+------------------------------------+
|                                    |
|         Topics / Field Tree        |
|     (原始數據樹狀圖，支援關鍵字搜尋)     |
|                                    |
+------------------------------------+
| ====== 繪圖快捷區 (Quick Plots) ===== |
|   [📈 振動量分析 (FFT)]             |
|   [🎯 控制器追隨響應]                |
|   [⚡ 馬達與電調回饋]                |
|   [🧲 磁場與航向角分析]              |
|   [🎛️ 飛行狀態與模式]                |
+------------------------------------+
```

### 3.1 繪圖快捷區 (Quick Plot Presets) 運作流程
1. **點選/拖曳觸發**：使用者可以將快捷區的分析套餐拖曳至空的 dashboard 畫框（或點擊該套餐）。
2. **延遲計算 (Deferred Computation)**：
   * 拖曳落入畫框之前，系統**完全不進行** FFT、插值對齊或 Norm 計算。
   * 拖入落定後，對應畫框轉為 Loading 狀態，主執行緒發送計算請求至 Web Worker。
   * Worker 計算完成後，回傳 TypedArrays（如 FFT 頻譜數值），圖表即時繪製出來。此設計保證了無效計算零佔用，前端絕不卡頓。

## 4. 模組化客製運算管理 (Modular Custom Calculations)

為了預留未來使用者導入自訂計算結構，系統設計了一套 JSON 格式的客製運算描述子 (Calculation Config Descriptor)。使用者可以透過 UI 上傳自訂的配置檔來宣告新的計算指標，系統將自動解析並新增至「自訂快捷區」。

### 4.1 自訂配置 JSON Schema 範例
```json
{
  "id": "custom-battery-power",
  "name": "電池瞬時功率計算",
  "description": "計算電壓與電流之乘積，評估瞬時耗能",
  "targetChart": {
    "title": "電池功率 (W)",
    "yLabel": "Power (W)",
    "color": "#fbbf24"
  },
  "dependencies": {
    "primary": {
      "topic": "battery_status",
      "fields": ["voltage_v", "current_a"]
    }
  },
  "operations": [
    {
      "type": "multiply",
      "inputs": [
        "battery_status.voltage_v",
        "battery_status.current_a"
      ],
      "output": "power_w"
    }
  ]
}
```

### 4.2 支援的運算子類型 (Supported Operations)
* **`norm`**: 計算三軸長度，$\sqrt{x^2 + y^2 + z^2}$。
* **`multiply` / `add` / `subtract` / `divide`**: 兩欄位或欄位與常數的基礎運算。
* **`fft`**: 時域轉頻域振幅。
* **`interpolate`**: 將資料 A 線性插值到與資料 B 對齊的時間軸。
* **`lowpass`**: 一階低通數位濾波，消除高頻雜訊。

### 4.3 客製化載入機制
1. 側邊欄新增「Import Config」按鈕。
2. 使用者上傳 JSON，前端進行 Schema 語法校正，確認 Topic 在 ULog 內皆存在。
3. 通過後，將該計算模組動態註冊到 React State。使用者拖曳它時，運算解析器會自動依 operations 鏈條在 Web Worker 中排程計算，最後渲染至 uPlot。

## 5. 系統檔案異動與開發時程 (File Changes & Dev Roadmap)

### 5.1 預計新增與修改檔案
* **`src/parser/fft.ts` (新增)**: 包含基頻、漢寧窗、FFT 幅值計算。
* **`src/parser/mathUtils.ts` (新增)**: 向量 Norm、插值對齊、常規數學運算子與時間戳平滑函數。
* **`src/parser/customCalcParser.ts` (新增)**: 負責讀取 custom JSON，解析運算鏈。
* **`src/components/VibrationPanel.tsx` (新增)**: 獨立呈現 Acc/Gyro 頻譜折線圖。
* **`src/components/PidResponsePanel.tsx` (新增)**: 提供 Rate/Att/Vel/Pos 切換分頁與對位圖表。
* **`src/components/MotorStatusPanel.tsx` (新增)**: 馬達 RPM、輸出與柱狀圖。
* **`src/components/MagneticPanel.tsx` (新增)**: 磁場模長與航向角比較。
* **`src/components/StatusModePanel.tsx` (新增)**: 飛行狀態、遙控器搖桿輸入、系統故障與安全防護時序呈現。
* **`src/components/Sidebar.tsx` (修改)**: 版面切為上下兩區，加入快捷繪圖與 JSON 匯入功能。

### 5.2 實作時程
1. **M1 (2天)**: 基礎數值核心與 FFT 模組實作（含單元測試與時間對齊）。
2. **M2 (2天)**: Web Worker 運算調度及自訂 JSON 解析器框架建置。
3. **M3 (3天)**: 五大進階分析面板（Vibration, PID, Motor, Mag, Status/Mode）UI 與 uPlot 繪圖對接。
4. **M4 (1天)**: 側邊欄快捷操作區及 UI 優化，完成驗收。
