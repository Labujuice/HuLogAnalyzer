# 專案需求與設計規格書：進階控制器分析 (維納反褶積階躍響應、執行器 FFT 頻譜與 PX4 多主題擴充) (2026-07-08)

## 1. 專案背景與擴充目標 (Overview)

為了提供更深度的飛控控制器診斷能力，本需求文件規劃了控制器分析工具 (Controller Analysis Tool) 的升級方案。主要目標為解決「若飛行中無刻意做出的突發階躍動作，則無法評估階躍響應」之痛點。

擴充目標包含：
1. **頻域維納反褶積 (Wiener Deconvolution)**：用於閉迴路系統識別 (System Identification)，使任何操縱動態下均能準確估算出階躍響應。
2. **執行器與反饋信號 FFT 頻譜分析**：用於偵測共振、噪聲與評估濾波器效果。
3. **PX4 診斷主題擴充**：整合軌跡指令源 (`trajectory_setpoint`)、起降狀態過濾 (`vehicle_land_detected`) 與執行器飽和判定。

## 2. 核心功能規格 (Technical Specifications)

### 2.1 頻域維納反褶積系統識別 (Wiener Deconvolution System ID)
將整段日誌（或使用者框選的時間區間）當作連續信號，透過系統識別演算法解算等效階躍響應。

#### (1) 物理系統模型
輸入信號為指令變化率 $u'(t)$（對 Setpoint 取微分），輸出信號為實測變化率 $y'(t)$（對 Actual 取微分）。兩者關係可建模為：
$$y'(t) = h(t) * u'(t) + n(t)$$
其中 $h(t)$ 為系統的脈衝響應 (Impulse Response)。

#### (2) 頻域解解算 (Deconvolution) 步驟
1. **微分與去直流**：
   * 計算 Setpoint 與 Actual 的一階差分（代表變化率，能更好地保留高頻暫態特徵並去除常規位移的直流偏置）：
     $$u_d[i] = \frac{u[i] - u[i-1]}{dt}, \quad y_d[i] = \frac{y[i] - y[i-1]}{dt}$$
2. **加窗處理 (Hanning Window)**：
   * 對 $u_d$ 與 $y_d$ 套用漢寧窗以降低頻域洩漏。
3. **快速傅立葉變換 (FFT)**：
   * 將時域差分信號轉換至頻域：
     $$U(f) = \text{FFT}(u_d), \quad Y(f) = \text{FFT}(y_d)$$
4. **維納濾波解褶積**：
   * 根據維納公式計算頻域傳遞函數估算值 $H(f)$：
     $$H(f) = \frac{Y(f) \cdot U^*(f)}{|U(f)|^2 + \text{SNR}^{-1}}$$
   * *註：$U^*(f)$ 為共軛複數；$\text{SNR}^{-1}$ 為噪聲-信號能量比（正則化常數，預設為 $0.01 \sim 0.1$，允許使用者於介面微調，用於防範除以零）*。
5. **逆傅立葉變換 (IFFT) 與積分**：
   * 對 $H(f)$ 進行逆傅立葉變換，取其實部得到時域脈衝響應 $h(t)$：
     $$h(t) = \text{Re}(\text{IFFT}(H(f)))$$
   * 對脈衝響應進行數值積分，獲得估算的正規化階躍響應 $s(t)$：
     $$s(t) = \sum_{k=0}^{t} h(k) \cdot dt$$

### 2.2 控制與執行器 FFT 頻譜分析 (Control & Actuator FFT)
分析率控制器輸出與反饋的頻率響應，判斷高頻共振與濾波衰減。

* **數據來源**：
  * Rate 控制輸出：`actuator_controls_0` 或 `actuator_motors`（輸出命令）。
  * 實際率反饋：`vehicle_angular_velocity`（實測角速度）。
* **算法要求**：
  * 去除直流分量、套用 Hanning Window。
  * 執行單軸 FFT，轉換為頻率 (Hz) vs 振幅幅值。
  * 頻率坐標軸限制在 $0 \sim F_s/2$ (奈奎斯特頻率，通常約為 $100 \text{ Hz} \sim 250 \text{ Hz}$)。

### 2.3 進階 PX4 主題整合 (Extended PX4 Topics Support)
擴充以下 Topic 以進行更細緻的診斷：

1. **軌跡源比對 (`trajectory_setpoint`)**：
   * 若日誌中包含 `trajectory_setpoint`，可用以取代 `vehicle_local_position_setpoint` 作為位置/速度環分析的期望值。
   * **目的**：評估輸入平滑（如 Jerk limits）產生的延遲與純 PID 追隨延遲的比例。
2. **飛行狀態過濾 (`vehicle_land_detected` & `vehicle_status`)**：
   * 讀取 `vehicle_land_detected.landed`（若為 `true` 則排除）與 `vehicle_status.nav_state`（僅在自動、位置、Offboard 等閉迴路模式下進行計算）。
   * **目的**：剔除地面怠速或手動模式下的無效干擾段。
3. **動態邊界標記 (`vehicle_constraints`)**：
   * 讀取最大速度限制限制 `speed_xy`、`speed_z`，在速度環對齊圖中以陰影或限制線標記，避免將「物理限速」誤判為「P 增益不足」。
4. **輸出飽和監測 (`actuator_outputs` / `actuator_motors`)**：
   * 監控各馬達輸出值是否持續達到上限值（例如 DShot 2000 或 PWM 2000）。若發生飽和，應在追隨報告中標註「Actuator Saturated (執行器飽和)」，提示硬件動力瓶頸。

## 3. 開發實作路徑與修改規範 (Implementation Guide for LLM Agents)

未來 LLM Agent 在實作此功能時，應遵循以下檔案架構進行開發：

### 3.1 數學運算層：`src/parser/mathUtils.ts`
新增以下數學工具函數：
1. **`wienerDeconvolve(input, output, dt, snr)`**：
   * 輸入：時域 `Float32Array` 輸入與輸出序列、採樣間隔、SNR 正則化參數。
   * 內部需依賴 `fft.ts` 進行 forward FFT 與 inverse FFT。
   * 輸出：估算出的 `Float32Array` 時域脈衝響應。
2. **`cumulativeIntegrate(series, dt)`**：
   * 輸入：脈衝響應序列。
   * 輸出：積分後的階躍響應數值。

### 3.2 背景線程層：`src/workers/ulogWorker.ts`
1. 新增指令類型 `CALC_WIENER_ID`。
2. 從 `parser` 實例提取指定 Topic 與 Field。
3. 利用 `vehicle_land_detected` 與 `vehicle_status` 對時間戳範圍進行 Slice，過濾出閉迴路飛行區間。
4. 調用 `wienerDeconvolve` 及 `cumulativeIntegrate`，並將生成的標準化階躍響應陣列轉移 (Transferable) 至主線程。

### 3.3 元件 UI 與圖表層：`src/components/PidResponsePanel.tsx`
1. **介面微調**：
   * 在階躍響應圖表上方新增一個切換控制列 (Segmented Control)：
     * `[時域滑動窗口法 (Time Window)]` (預設)
     * `[維納反褶積法 (Wiener ID)]`
   * 若選擇 `維納反褶積法`，顯示一個 SNR 滑桿（範圍 $10 \sim 1000$，對應正則化參數 $\text{SNR}^{-1}$ 倒數）。
2. **FFT 頻譜展示**：
   * 在「時域對齊比較圖」與「正規化階躍響應圖」下方，新增第三張小圖表：「執行器控制與反饋頻譜 (Control/Feedback FFT Spectrum)」，顯示 Rate 控制命令與實際角速度在選定區間內的頻譜強度疊合對比。
3. **PX4 Topic 預設適配**：
   * 更新 `LOOP_CONFIGS`，當 `trajectory_setpoint` 存在時，優先將其作為位置/速度環的 Setpoint 數據源供使用者切換。

## 4. 驗證標準與驗證案例 (Verification & Test Cases)

1. **零激發測試**：若所選時間內飛機完全靜止或無操作（Setpoint 均為常數），切換至「維納反褶積法」時，UI 應優雅地顯示「無足夠激發信號，無法進行系統識別」，而非崩潰或輸出隨機雜訊。
2. **極限限制驗證**：在大範圍階躍輸入時，若馬達輸出達到最大值，診斷區塊應能顯示馬達飽和警告，且能正確讀取並顯示 `vehicle_constraints` 限制。
3. **離線雙擊啟動合規性**：所有新增的圖表庫調用或相對路徑調用必須遵守專案的 `AGENTS.md` 規範，全部使用**相對路徑**，確保 `dist/index.html` 雙擊即可離線執行。
